API Documentation

The Adaptive Auth API provides AI-powered login risk assessment. Send login context data and receive a real-time risk score with an actionable decision: ALLOW, MFA, or BLOCK.

💡 Base URL: http://localhost:5000 (local) or your deployed domain. All API responses are JSON.

Authentication

All API requests require an API key sent via the X-API-Key header.

Header
X-API-Key: aa_live_your_key_here

Get your API key by signing up (free tier: 500 calls/month) or from the API Dashboard.

Quick Start

Here's the fastest way to make your first API call:

cURL
curl -X POST http://localhost:5000/api/v1/assess \
  -H "X-API-Key: aa_live_your_key_here" \
  -H "Content-Type: application/json" \
  -d '{"country":"India","device_type":"desktop","hour_of_day":14,"threat_score":10}'

Risk Assessment Endpoint

POST /api/v1/assess

Assess the risk level of a login attempt. Returns a risk score (0.0–1.0) and a recommended action.

Request Body (JSON)

Field Type Required Description
country string optional Country name, e.g. "India", "Russia". Default: "Unknown"
region string optional Region/continent, e.g. "South Asia". Default: "Unknown"
hour_of_day integer optional Hour of login (0–23). Default: current UTC hour
device_type string optional One of: "desktop", "mobile", "tablet", "iot_device", "smart_tv". Default: "desktop"
prev_login_success integer optional Was the user's last login successful? 0 or 1. Default: 1
threat_score integer optional IP reputation score (0–100). Higher = more suspicious. Default: 0
distance_from_last_login float optional Distance in km from user's last login location. Default: 0

Response Format

200 OK — Successful Assessment
{
  "risk_score": 0.8234,
  "action": "BLOCK",
  "reasons": ["high_threat_score", "unusual_distance", "high_risk_country"],
  "request_id": "req_a1b2c3d4e5f67890",
  "usage": {
    "calls_today": 42,
    "calls_this_month": 156,
    "monthly_limit": 500
  }
}
Field Type Description
risk_score float Risk probability from 0.0 (safe) to 1.0 (dangerous)
action string ALLOW (<0.3), MFA (0.3–0.7), or BLOCK (>0.7)
reasons array Risk factors detected (e.g., "high_threat_score", "unusual_hour")
request_id string Unique ID for this assessment request
usage object Your current API usage (calls today, this month, and monthly limit)

Possible Risk Reasons

Reason Trigger
high_threat_score threat_score > 60
unusual_distance distance_from_last_login > 3,000 km
unusual_hour hour_of_day < 5 or > 23
unusual_device device_type is "iot_device" or "smart_tv"
high_risk_country country is North Korea, Iran, or Russia
no_previous_success prev_login_success is 0

Error Codes

HTTP Status Code Description
401 AUTH_REQUIRED Missing X-API-Key header
401 INVALID_KEY API key not found or revoked
429 LIMIT_EXCEEDED Monthly API call limit reached — upgrade your plan

Python — requests
import requests

response = requests.post(
    "http://localhost:5000/api/v1/assess",
    headers={"X-API-Key": "aa_live_your_key_here"},
    json={
        "country": "Russia",
        "region": "Eastern Europe",
        "hour_of_day": 3,
        "device_type": "mobile",
        "prev_login_success": 0,
        "threat_score": 85,
        "distance_from_last_login": 5000
    }
)

data = response.json()
print(f"Risk: {data['risk_score']} → {data['action']}")
print(f"Reasons: {data['reasons']}")

# Based on the action, decide what to do:
if data["action"] == "ALLOW":
    print("✅ Let the user in")
elif data["action"] == "MFA":
    print("🔐 Require MFA verification")
else:
    print("🚫 Block the login attempt")
JavaScript — fetch
const response = await fetch("http://localhost:5000/api/v1/assess", {
  method: "POST",
  headers: {
    "X-API-Key": "aa_live_your_key_here",
    "Content-Type": "application/json"
  },
  body: JSON.stringify({
    country: "Russia",
    device_type: "mobile",
    hour_of_day: 3,
    threat_score: 85,
    distance_from_last_login: 5000
  })
});

const data = await response.json();
console.log(`Risk: ${data.risk_score} → ${data.action}`);

// Handle the decision
switch (data.action) {
  case "ALLOW":  /* proceed with login */   break;
  case "MFA":    /* request MFA code */     break;
  case "BLOCK":  /* deny the login */       break;
}
